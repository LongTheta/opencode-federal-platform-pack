# v0.3.0 Release Notes — 10/10 Grade

**Goal:** Address all gaps from the B (8.2/10) assessment to reach 10/10.

---

## Changes

### 1. Runtime Schema Validation

- **scripts/validate-schema.js** — Validates JSON against review-score or quality-gate schema
- **npm run validate:review-score** / **validate:quality-gate** — Convenience scripts
- **Dependencies:** ajv, ajv-formats

### 2. Configurable Push Blocking

- **FEDERAL_PLATFORM_BLOCK_PUSH=1** — Plugin blocks `git push` until `/quality-gate` is run
- Default: warn only (unchanged)
- Documented in enforcement-model.md, INSTALL.md, README

### 3. Native Tool Implementations

- **scripts/review-score.js** — Computes weighted score, letter grade, production readiness from category scores
- **scripts/quality-gate-check.js** — Returns verdict (pass/pass_with_warnings/fail) from blockers/warnings
- Both accept stdin or file path; exit code 2 for quality-gate fail

### 4. CI Integration

- **.github/workflows/quality-gate.yml** — Runs on PR and push to main/master
- Steps: checkout, npm install, npm run verify, schema validation

### 5. Smoke Test / Verification

- **scripts/verify-pack.js** — Verifies structure, opencode.json, schema validation, review-score and quality-gate-check scripts
- **npm run verify** — Runs full verification

### 6. Schema Consolidation

- **platform-review-report.json**, **review-report.json** — Deprecated (x-deprecated, x-replace-with)
- **schemas/README.md** — Canonical vs deprecated schema list

---

## Upgrade

```bash
npm install
npm run verify
```

Set `FEDERAL_PLATFORM_BLOCK_PUSH=1` in your environment to block push without quality-gate.
